# 1. Составить и правильно оформить следующие тестовые артефакты: тест-план, чек-листы и тест-кейсы
## Тест план
### Введение
#### Исходные данные:
Страница контактов пользователя, доступная как через web браузер, так и из гибридного мобильного приложения для Android и iOS.
### Следующие компоненты и функции будут протестированы: 
1. Страница с контактами пользователя
2. Страница редактирования данных пользователя
### Стратегия тестирования: 
1. Тестирование верстки с выявлением и описанием дефектов
2. Функциональные тесты, тестирование кроссбраузерности с выявлением и описанием дефектов
3. Тестирование на мобильных устройствах с выявлением и описанием дефектов
#### Платформы, утвержденные к проверке:
1. Desktop
2. Android (7+)
3. iOS (11.0+)
#### Браузеры, утвержденные к проверке:
1. Google Chrome (latest)
2. Firefox (latest)
3. Opera (latest)
4. Internet Explorer 11
5. Internet Explorer 10
6. Internet Explorer 9
7. Safari (latest)
### Типы тестирования:
1. Функциональное тестирование: 
- Проверка корректности отображаемых данных на всех страницах. 
- Проверка редактирования данных путем реализации тестовых сценариев. 
2. Тестирование верстки: 
- Проверка страниц на соответствие дизайну, корректную работу полей ввода и элементов управления
3. Тестирование кроссбраузерности: 
- Проверка верстки и функционала в различных браузерах
4. Тестирование на мобильных устройствах:
- Проверка верстки и функционала на различных мобильных устройствах.

## Чек листы
### Функциональное тестирование
#### Страница контактов пользователя 
- Отображение данных пользователя 
- Кликабельные ссылки открывают соответствующие приложения/сайты
#### Страница редактирования данных пользователя 
- Поле "имя" не принимает цифры и символы
- Поле "фамилия" не принимает цифры и символы
- Поле "основной телефон" не принимает буквы и символы (кроме +)
- Валидация поля "основная почта"
- "крестик" для очищения поля ввода на всех инпутах
- Добавление "контакта"
- Можно добавить тот же "контакт" несколько раз (два скайпа например) 
- Удаление "контакта" (значок корзины)
- Валидация заполнения обязательных полей
- Сохранение данных 
- Отмена 
- Заменить фото 
- Удалить фото
- Закрытие модального окна по нажатию на крестик
### Тестирование верстки
- Соответствует макету
- Адаптивность (Корректное отображение на всех требуемых размерах окна браузера)
- Отображение страницы при масштабировании в десктопных браузерах
- Корректное масштабирование фотографии
- Нет ошибок JS в консоли браузера
- Скролл отображается только при необходимости
### Тестирование кроссбраузерности
- Корректная работа при включенном расширением AdBlock в браузере
- Корректное отображение кнопок, полей ввода, выпадающего списка в разных браузерах
### Тестирование на мобильных устройствах
- Корректная работа на разных устройствах
- Корректное отображение кнопок, полей ввода, выпадающего списка на разных устройствах

## Тест кейсы 
### Профиль 
#### Клик по почте
- Нажать на значение поля почта
- Ожидаемый результат - открывается соответствующее приложение
#### Клик по карандашу
- Нажать на карандаш
- Ожидаемый результат - открывается модальное окно редактирования профиля

### Редактирование профиля 
#### Редактирование полей
- Открыть редактирование профиля
- Изменить значение в заполненном поле 
- Нажать кнопку "Сохранить"
- Ожидаемый результат - модальное окно закрылось, в профиле значение, соответствующее отредактированному полю, изменилось
#### Добавление полей
- Открыть редактирование профиля
- Нажать на "Добавить контакт"
- Выбрать контакт из выпадающего списка 
- Ожидаемый результат - соответствующий блок отображается на странице редактирования
#### Позитивная проверка валидации обязательных полей
- Открыть редактирование профиля
- Удалить значения всех полей, кроме "Имя"
- Нажать кнопку "Сохранить"
- Ожидаемый результат - изменения сохранены и отображаются в профиле
#### Негативная проверка валидации обязательных полей
- Открыть редактирование профиля
- Удалить значение поля "Имя"
- Нажать кнопку "Сохранить"
- Ожидаемый результат - сообщение об ошибке
#### Отмена редактирования
- Открыть редактирование профиля
- Изменить значения некоторых полей 
- Нажать кнопку "Отмена"
- Ожидаемый результат - модальное окно закрылось, данные в профиле не изменились
#### Закрытие по крестику
- Открыть редактирование профиля
- Нажать крестик
- Ожидаемый результат - модальное окно закрылось
